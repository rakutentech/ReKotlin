Dispatch an Action or Effect to the store.

An Action is the simplest way to initiate state changes. This passes the action to Middleware, delegates to the Reducer to determine the new state. and eventually notifies Subscribers of (sub-)state changes (if any).

Effects are one time events that do not change the state. This notifies all Effect listeners.

This will not pass anything to Middleware, delegate to Reducer or change state.

Example of dispatching an action:

// dispatch an action
store.dispatch(CounterAction.IncreaseCounter)
// dispatch an effect
store.dispatch(AnimationEffect

Parameters

dispatchable

an action or effect to dispatch to the store
